## Not Yet Supported

These features are recognized but not implemented. Workflows using them will either
be partially executed or fail at parse time.

| Feature | Notes |
|---|---|
| `uses:` actions | Marketplace and local actions are not supported |
| `matrix:` strategy | No matrix expansion |
| `container:` / `services:` | No Docker containers |
| Reusable workflows | `workflow_call` not supported |
| Composite actions | Not supported |
| `$GITHUB_ENV` | Cannot set env vars for subsequent steps via file |
| `$GITHUB_PATH` | Cannot modify PATH for subsequent steps via file |
| Full expression syntax | Only `steps.*.outputs.*` patterns are expanded |
| `env.*` / `github.*` context | Expression contexts beyond step outputs |
| Functions in expressions | `contains()`, `toJSON()`, `startsWith()`, etc. |
| Secrets | `secrets.*` context |
| Permissions | `permissions:` field |
| Concurrency | `concurrency:` groups |
| Timeout | `timeout-minutes:` |
| Artifacts | `actions/upload-artifact`, `actions/download-artifact` |

## Roadmap

Planned for future versions:

1. **`$GITHUB_ENV`** and **`$GITHUB_PATH`** support
2. **Full expression evaluation** (`env.*`, `github.*`, functions)
3. **MicroVM isolation** — run jobs inside CloudHypervisor VMs
4. **`uses:` actions** — clone and execute action repositories
